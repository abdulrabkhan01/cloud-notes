What is Cloud Computing

Usually every company is having its own data center, but creating and managing the datacenter, servers and all hardware is an extra burden for the organizations, so cloud computing solves this problem by providing these datacenter and hardward capabilities as a service.

Cloud Computing Providers

Following are some of the cloud computing Providers
  • AWS by Amazon
  • GCP by Google
  • Azure by Microsoft
  • OCP by Oracle

What is AWS

AWS stands for Amazon web services which provide different cloud computing services, it is one of the leading cloud computing provider.

  • Public Cloud - Accessible over internet
  • Private Cloud - Accessible over VPN
  • Hybrid Cloud - Combination of Public and Private Clouds

Scaling concepts

  • Scaling up - Adding more resources to a single VM or instance, eg. Increase RAM capacity
  • Scaling down -Reducing the size of a resource in single VM or instance, eg. Decrease RAM
  • Scaling out - Increasing the number of VMs to make them work in parallel to distribute the high traffic
  • Scaling in - Decreaing the number of VMs when load decreases
  • Autoscaling - Automatically handling scaling out/in using rules/policies etc.

What is Server Less Architecture

It is a way to build and run applications without having to manage the Infrastructure, this comes under Software as a Service concept.

Regions and Availability Zones

  • Region: Physical Geographical location
  • Availability Zones: Data center inside a particular region

3 AWS - S3 (Simple Storage Service)

AWS S3 is highly secure, scalable & durable Object Storage Service

S3 is Object based, it uses Key/Value Pairs for storing the objects

  • Key: Name of the Object
  • Value: Actual File Data in bytes format
  • Version Id: Used for versioning
  • Metadata: Data related to the object stored

Consistency for S3 Objects

  • PUT: Read after Write Consistency ( able to read file immediately after storing to S3 bucket)
  • Overwrite PUT & DELETE: Eventual Consistency

S3 Guarantee

  • Availability: 99.9% Guarantee
  • Durability: 99.999999999% Guarantee (11 9s)

S3 Storage Classes

  • S3 Standard: 99.99% Availability + 99.999999999% Durability + Redundancy (Sustain the loss of 2 facilities at the same time)
  • S3 IA (Infrequently Accessed): For Infrequently accessed data that should be available immediately when access, lower fee than Standard, but retrieval fee applies
  • S3 One Zone IA (Infrequently Accessed): Infrequently Accessed Data but only for single Availability Zone.
  • S3 Intelligent Tiering: Automatically switch from higher cost to lower costs without impacting processing or performance.
  • S3 Glacior: Low Cost Storage for less frequenty accessed data with retrieval times in minutes to hours.
  • S3 Glacior Deep Archive: Low Cost Storage for less frequenty accessed data with retrieval times 12 hours acceptable.

S3 Restricting Bucket Access

  • Use Bucket Policies: These are the policies applied at the bucket level.
  • Use Object Policies: These are the policies applied at the individual object level.
  • IAM Policies: Group/User Level IAM policies

S3 Security & Encryption

Once S3 bukcet is created it is private by default, to setup access control use any of the below
  • Bucket Policies
  • Access Control List (ACL)
Encryption at Rest is achieved using the following options:
  • S3 Managed Keys
  • AWS Key management service
  • Server Side Encryption using Customer Provided Keys

S3 Versioning

  • All versions are stored and even the deleted ones also.
  • Once enabled you cannot disabled it, but only suspend

S3 Lifecycle Rules

  • Can be used to automate the movement from one storage tier to another.
  • Can be used in conjunction of versioning
  • Can be applied to all versions

4 AWS - EC2 (Elastic Compute Cloud)

EC2 is AWS Service used to provide resizable compute instances within minutes. Allowing to quickly Scale up and down.

EC2 Pricing Models

  • On Demand: Fixed Rate by hour or seconds with no committment.
  • Reserved: Provided capacity reservation, discount on hourly charge per instance, Contract Term 1 or 3 years.
  • Spot: On the spot, allows to bid for the price with greater saving, used when application has flexible start/end times
  • Dedicated Hosts: Physcical Servers dedicated for your use, possible to use your own licence and save costs.

5 Differnt Types of EC2 instances

  • General Purpose: Balance of computer memory, storage and network resources (Business critical, mid size database, web tiered applications.
  • Compute Optimized: Used for high performance computing like batch processing, video encoding etc.
  • Memory Optimized: Used for high performance database, distributed in memory caches, big data analytics etc.
  • Accelerating Computing: Machine Learning, Graphic Intensive Apps, Games etc.
  • Storage Optimized: No SQL Database, data warehousing, distributed file systems.

6 AWS Security Groups

Used to control traffic to the AWS Resources, acts as a Virtual Firewall. Once you associate Security Group to any EC2 instance it will handle all inbound and outbound traffic rules.
  • All Inbound traffic blocked by default
  • All Outbound traffic is allowed
  • Changes to Security Group take effect immediately
  • No restriction on number of EC2 instances in a Security Group
  • Also possible to attach multiple security groups to a singel EC2 instance
  • Security Groups are stateful
  • Security Groups cannot be used to block specific IP Address, use ACL(Access Control Lists) for it
